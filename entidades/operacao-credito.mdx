Uma **Operação** representa uma análise de crédito de um borderô CNAB 400. Cada operação passa por filas de processamento e produz um resultado hierárquico com regras aplicadas.

## Ciclo de vida

```
criada → queued → processing → completed / failed
```

## Campos principais

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `operationId` | UUID | Identificador único da operação |
| `status` | enum | `queued`, `processing`, `completed`, `failed` |
| `engineId` | UUID | Motor de regras utilizado |
| `fileName` | string | Nome do arquivo CNAB enviado |
| `tituloCount` | number | Total de títulos processados |
| `approvedCount` | number | Títulos aprovados |
| `blockedCount` | number | Títulos bloqueados |
| `alertCount` | number | Títulos com alerta |

## Resultado da análise

O resultado segue uma árvore hierárquica:

```
Cedente (empresa que cede os títulos)
  └── Sacados (devedores)
       └── Títulos (duplicatas/boletos)
            └── Issues (regras violadas)
```

### Status de cada entidade

- **approved** — Sem violações
- **blocked** — Regra bloqueante violada
- **alerted** — Regra de alerta violada

### Issues (violações)

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `ruleId` | string | ID da regra |
| `ruleName` | string | Nome da regra |
| `category` | enum | `blocking` ou `alert` |
| `message` | string | Mensagem descritiva |
| `detail.field` | string | Campo avaliado |
| `detail.actual` | any | Valor encontrado |
| `detail.expected` | any | Valor esperado |
| `dataSource` | string | Fonte dos dados (ex: "XML NFe", "Sherlocker NFe") |

### Summary

| Campo | Descrição |
|-------|-----------|
| `totalTitulos` | Total de títulos |
| `approved` / `blocked` / `alerted` | Contagem por status |
| `totalValue` / `approvedValue` / `blockedValue` / `alertedValue` | Valores monetários |
| `sacados.total` / `sacados.withIssues` | Contagem de sacados |
