Um **Imovel** no Sherlocker representa um registro de propriedade, cobrindo tanto imoveis urbanos (via portais IPTU municipais) quanto rurais (SNCR, CAFIR, IBAMA).

## Pontos de entrada

| Dado disponivel | Endpoint | Resultado |
|----------------|----------|-----------|
| CPF | `GET /pessoas/cpf/{cpf}/imoveis` | Imoveis urbanos da pessoa |
| CPF | `GET /pessoas/cpf/{cpf}/rural` | Perfil rural completo |
| CNPJ | `GET /empresas/cnpj/{cnpj}/imoveis` | Imoveis da empresa |
| CNPJ | `GET /empresas/cnpj/{cnpj}/rural` | Rural da empresa |
| CPF/UF | `GET /imoveis/cpf/{cpf}/{uf}` | Filtrado por estado |

<Tip>
  Na rota principal de pessoa (`GET /pessoas/cpf/{cpf}`), os imoveis ja vem incluidos nos campos `imoveis` (urbanos) e `rural`. Use as sub-rotas para consultas mais rapidas.
</Tip>

## Imoveis urbanos

Cobertura de **1.400+ municipios** via integracao com portais IPTU.

### Estrutura

| Campo | Descricao | Exemplo |
|-------|-----------|---------|
| `id` | Identificador unico | `a1b2c3d4e5f6` |
| `fonte` | Portal de origem | `bh`, `sp-iptu`, `poa` |
| `sql_iptu` | Inscricao municipal | `0123456789` |
| `localizacao.logradouro` | Rua/Avenida | `R HENRIQUE PASSINI` |
| `localizacao.numero` | Numero | `748` |
| `localizacao.bairro` | Bairro | `SERRA` |
| `localizacao.cidade` | Cidade | `BELO HORIZONTE` |
| `localizacao.uf` | Estado | `MG` |
| `localizacao.cep` | CEP | `30220-380` |
| `area_terreno` | Area do terreno (m2) | `360.00` |
| `area_construida` | Area construida (m2) | `180.00` |
| `ano_construcao` | Ano de construcao | `1995` |
| `tipo_uso` | Uso do imovel | `RESIDENCIAL`, `COMERCIAL` |
| `valor_venal` | Valor venal (R$) | `450000.00` |
| `valor_iptu` | Valor do IPTU anual (R$) | `2400.00` |

### Proprietario

Cada imovel inclui dados do proprietario:
- `nome` — nome conforme cadastro municipal
- `documento` — CPF ou CNPJ
- `tipoDocumento` — `CPF` ou `CNPJ`

## Imoveis rurais

O perfil rural (`/cpf/{cpf}/rural`) agrega dados de multiplas fontes:

| Fonte | Descricao |
|-------|-----------|
| **SNCR** | Sistema Nacional de Cadastro Rural (INCRA) — area, modulos fiscais, detentores |
| **CAFIR** | Cadastro Fiscal de Imoveis Rurais (Receita Federal) — NIRF, situacao fiscal |
| **IBAMA** | Fiscalizacao ambiental — multas, embargos, areas embargadas |
| **DOF** | Documento de Origem Florestal — transporte de produtos madeireiros |
| **Produtores** | Cadastro de produtores organicos |

## Exemplo de uso

```python
import requests

BASE = "https://221b-api.sherlocker.com.br/api/v1"

# Imoveis urbanos via perfil completo
pessoa = requests.get(f"{BASE}/pessoas/cpf/12345678901", params={"token": TOKEN}).json()
imoveis = pessoa.get("imoveis", [])
rural = pessoa.get("rural", {})

# Via sub-rota especifica
resp = requests.get(f"{BASE}/pessoas/cpf/12345678901/imoveis", params={"token": TOKEN}).json()

for im in imoveis:
    loc = im.get("localizacao", {})
    print(f"{loc.get('logradouro')}, {loc.get('numero')} - {loc.get('cidade')}/{loc.get('uf')}")
    print(f"  Valor venal: R$ {im.get('valor_venal', 0):,.2f}")
    print(f"  Area: {im.get('area_terreno', 0)} m2 terreno, {im.get('area_construida', 0)} m2 construido")
```

## Estrategia de levantamento patrimonial

1. **Pessoa alvo** — buscar imoveis urbanos + rurais por CPF
2. **Parentes** — repetir para conjuge e filhos (imoveis podem estar em nome de familiares)
3. **Empresas** — buscar imoveis dos CNPJs onde a pessoa e socia
4. **Cruzamento** — comparar enderecos de moradia com enderecos de propriedade

<Info>
  Consultas de imoveis podem levar alguns segundos por dependerem de portais municipais. Para consultas em lote, use `?enrich=true` para incluir dados de leilao e Caixa Economica.
</Info>
